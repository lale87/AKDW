\subsection{DOM-Basiert (lokal)}
%----------------------------------
DOM-Basiertes XSS nutzt Url-Parameter, um Schadcode auf einem fremden Client zur
Ausführung zu bringen. Die Besonderheit in dieser Methode liegt darin, dass der
im Url-parameter eingebettete Schadcode nicht vom Server ausgelesen und in die
Webseite eingebettet wird, sondern erst durch den Client. 
Das Einbetten des Schadcodes in die Webseite erfolgt dadurch, dass
Schwachstellen in dem unveränderten Script der Webseite ausgenutzt werden. Die
mit der Url gelieferten Parameter werden von dem Script der Webseite ausgelesen
und für die Darstellung verwendet. Wird nun eine Script-Routine als
Parameter übergeben und der Parameter wird durch das Script der Webseite
ungefiltert zur Anzeige gebracht, so wird das Script auf dem Client
ausgeführt. \\
\texttt{http://www.trustedsite.tld/welcome\#name=<script>alert('HACKED')</script>}\\
